## What's New

This release focuses on review quality. MistratonReview now skips files that do not benefit from AI review and uses severity indicators to make findings easier to scan.

### Features

- **Smart file filtering** - automatically skips lockfiles (`package-lock.json`, `pnpm-lock.yaml`), minified files, images, generated code, and TypeScript declaration files (`.d.ts`)
- **Severity-based indicators** in inline comments to distinguish critical bugs from warnings and suggestions
- **Improved diff position mapping** for accurate inline comment placement even on complex multi-hunk diffs
- **Reduced noise** by focusing the review on files where the AI adds real value

### Why This Matters

Reviewing generated or dependency files produces false positives and dilutes the signal from the AI. By filtering these files before sending the diff to Mistral AI, each review is more focused and the token usage is lower, which also reduces latency.

The severity levels make it easy to triage reviews at a glance - a pull request with only suggestions is safe to merge, while one with critical findings needs attention before approval.
