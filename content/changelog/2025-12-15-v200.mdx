## What's New

v2.0 is a reliability and performance overhaul. Large pull requests now review faster, webhook processing is more robust, and the Mistral AI client is initialized once instead of per-request.

### Features

- **Batched file review** - files are processed in parallel with a concurrency limit to avoid overwhelming the Mistral API on large PRs
- **Maximum 25 files per review** - enforces a hard cap so reviews complete in predictable time even on massive changesets
- **Mistral client singleton** - the Mistral AI client is created once at module load time and reused across all webhook invocations, reducing cold-start overhead
- **Improved webhook signature verification** - stricter HMAC validation with early rejection of malformed payloads

### Bug Fixes

- Fixed inline comment positioning drift on PRs with deleted lines
- Resolved double-review on synchronized push events
- Corrected error handling when Mistral returns a rate-limit response

### How the Batching Works

Each changed file is added to a queue. The processor pulls up to 5 files at a time and sends them to Mistral AI concurrently. Results are collected and posted as inline comments after all batches complete. This keeps total review time under 30 seconds for most pull requests while respecting API rate limits.
